# 随机生成器工作台（公开版）

这是一个面向角色设定与人物原型构建的前端工作台，目标不是只给出零散标签，而是把“人物背景、性格档案、人物外观”放进同一条生成链里，让使用者能够更快地得到一份结构完整、阅读顺畅、可以继续手动微调的人物卡。整个项目采用纯前端形式组织，打开页面即可运行，不依赖数据库或后端服务，适合本地使用、演示展示、教学说明，以及作为后续角色生成器项目的界面原型。

这套公开版保留了工作台最核心的生成体验：统一的顶部导航、统一的卡片式结果区、模块级重刷、字段手选、锁定卡位、串联生成、JSON 导出，以及围绕“当前人物槽位”展开的连续编辑流程。相比很多只给出一串随机字段的工具，这个工作台更强调“档案感”和“可继续编辑”，因此生成结果会被拆成多个模块，并允许逐字段修改，而不是一次生成后只能整张推倒重来。

## 项目定位

这个仓库适合以下几类使用场景：

- 为小说、剧本、互动叙事、游戏角色建立初始人物档案
- 为世界观设定或角色库搭建一套可快速浏览和对比的人物生成界面
- 作为角色生成器、设定工具、前端配置面板的参考实现
- 作为本地静态网页项目示例，展示多面板工具如何统一视觉与交互

工作台强调的不是“无限复杂的设定树”，而是“先快速得到一位相对可信的人物，再围绕重点字段继续微调”。这也是为什么页面结构会把结果卡做成模块网格，并把常见动作集中在卡片头和字段右侧，而不是把所有控制都塞进顶部表单区。

## 主要功能

当前公开版包含三条核心人物生成链路：

### 1. 人物背景

背景生成器负责给人物建立底盘，包括年龄段、性别认同、城市层级、常住城市、成长地区、教育状态、专业方向、职业大类、生活阶段、居住状态、家庭环境、语言习惯与社交密度等字段。它不是简单地从若干列表里各抽一个值，而是会依据年龄、教育状态、职业组别、地区规则等进行联动，因此结果更像一张可读的人物背景卡，而不是拼接式标签堆。

### 2. 性格档案

性格面板会输出核心维度、MBTI 派生、人际面向与桥接层。它既可以单独生成，也可以读取当前背景人物的部分条件，在同一人物槽位上继续往下补全。结果区按照模块拆分，便于快速浏览不同侧面，也方便做局部重刷，例如只改情绪模式、不动社交能量，或者只重做桥接层。

### 3. 人物外观

人物外观生成器覆盖整体轮廓、面部结构、口腔、腹臀腿、手足等模块。设计上它更像“身体外观档案”而不是一段自由文本，因此更适合做可视化卡片，也更方便作为后续图像生成、人物设定卡、立绘提示词的上游输入。模块化结构让外观不会被堆成长卡，阅读成本更低。

## 统一工作流

这套工作台的一大重点是“统一工作流”。在页面顶部，你可以直接使用 `串联生成`，让系统按固定顺序在同一人物槽位上依次补全：

- 人物背景
- 性格档案
- 人物外观

这样做的价值在于，使用者不需要自己记住“我刚才选的是哪一张卡”，也不用在不同面板之间手动同步。工作台会维护当前人物槽位，并把选择、锁定、导出这些动作围绕同一人物展开。对于需要快速做一批人物雏形的人，这种流程会比逐页单独操作顺手很多。

## 可编辑与可回滚

为了避免“随机一次不满意就只能整页重来”，工作台保留了几种很实用的编辑方式：

- 模块级重刷：只重做某个模块，不影响无关模块
- 字段手选：点击字段值后可直接切换到编辑态
- 卡位锁定：锁定后再次生成不会覆盖当前结果
- 对比上次：可以展开查看同一槽位前后差异
- JSON 导出：便于保存、整理、喂给其他流程继续处理

这几类动作配合在一起，能把“随机生成器”从一次性工具，提升成一个可以持续打磨人物的工作台。

## 关于公开版与 NSFW 设定

公开版默认关闭 NSFW 相关设定，因此首页只突出背景、性格与人物外观三条主线。私密部位与偏好档案仍然保留在界面结构中，但默认不展开；如果确有需要，可以通过顶部的 `设定私密部位与偏好` 按钮临时显示。

## 本地使用

这个项目是纯静态前端目录，使用方式非常直接：

1. 克隆或下载本仓库
2. 直接打开 `index.html`
3. 或者使用任意本地静态服务器预览

项目中的数据文件与生成器逻辑都已经放在同一目录下，因此这份公开版可以独立运行，不依赖父目录中的额外文件。

## 目录结构

- `index.html`：页面入口与面板结构
- `generator_workspace.css`：统一浅色主题、卡片、结果网格与响应式样式
- `generator_workspace_ui.js`：顶部操作、卡片增强、导出、串联、字段编辑代理
- `generator_workspace.js`：人物背景生成逻辑
- `personality_workspace.js`：性格档案生成逻辑
- `body_appearance_workspace.js`：人物外观生成逻辑
- `*_data.js`：各生成器使用的本地数据源
